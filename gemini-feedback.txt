### Technical Refinements for Your Spec

**1. Refine the Data Model (Traceability)**
In your `POLICY_UPDATES` table, consider adding a relationship to `ELIGIBILITY_RULES`.

* *Why:* When a Lender Letter changes the DTI limit, you want the system to flag *exactly* which row in your Rules Database is now obsolete.
* *Add:* `affected_rule_ids` (Array of UUIDs linking to `ELIGIBILITY_RULES`).

**2. The "Agentic" Loop for Fixes**
The "Check My Loan" feature requires a specific loop pattern. It is not a straight chain; it is a **ReAct (Reason + Act)** loop.

Here is how the **Scenario Reasoner** should function to ensure the "Fixes" are accurate:

1. **Evaluate:** Check loan inputs against stored Rule Set (HomeReady).
2. **Identify Gap:** "DTI is 52%, Limit is 50%."
3. **Reason:** "I need to lower DTI by 2%. I can either increase income or decrease debt."
4. **Tool Use (Calculator):** "Calculate required debt reduction: `(TotalDebt / Income) = 0.52` -> `TargetDebt = 0.50 * Income`. Difference = `TotalDebt - TargetDebt`."
5. **Verify:** Run the new numbers against the rule set again to confirm pass.
6. **Output:** Present the fix.
